Desafio 2 - Adoção de Logs Estruturados (formato JSON)
A chave para um monitoramento eficiente de APIs reside na estrutura dos logs ,
permitindo uma análise rápida e detalhada do comportamento do sistema.
Logs em formato de texto simples, como os exemplos a seguir , apresentam sérias limitações para a observabil idade:
Projeto Final: Desenvolvendo e Publicando uma API
2024 -01-15 10:30:45 INFO Login bem -sucedido: admin .
2024 -01-15 10:30:46 ERROR Erro na predicao: modelo nao carregado .
2024 -01-15 10:30:47 INFO Predicao: setosa para user admin .
Dificuldades
• Filtro Complexo: é difícil isolar eventos específicos (e.g., todos os erros de um usuário).
• Agregação Limitada: não é simples calcular métricas (e.g., latência média por endpoint).
• Correlação Ausente: não há um meio fácil de rastrear a sequência de eventos de uma única requ isição.
A adoção de logs em formato JSON transforma os dados brutos em informações ricas e pesquisáveis:
{
 "timestamp": "2024 -01-15T10:30:45.123Z",
 "level": "INFO",
 "service": "api -iris-v2",
 "trace_id": "abc12345",
 "message": "prediction_complete d",
 "user": "admin",
 "classe": "setosa",
 "confianca": 0.98,
 "latency_ms": 12.5
}

Vantagens do Log Estruturado
• Filtro Eficaz: use campos nomeados, como jq 'select(.user == "admin")' ,
para buscas precisas.
• Agregação Poderosa: utilize consultas baseadas em campos para métricas: SELECT AVG(latency_ms) FROM logs .
• Correlação Simples: o uso do trace_id permite rastrear todo o fluxo de uma requisição: WHERE trace_id = "abc12345".

Boas práticas - Modularizando o main.py e a documentação!
O arquivo main.py da v2 tornou -se excessivamente complexo. O problema é que ele centrali zava diversas responsabilidades, como configuração da aplicação ,
definição de schemas Pydantic , carregamento do modelo e definição de rotas .
Em projetos reais, essa centralização é um grande obstáculo, dificultando a localização do código, a evolução de no vas funcionalidades e a criação de testes.
Benefícios da Modularização
A modularização resolve esses problemas e traz as seguintes vantagens:
• Separação de Responsabilidades (SRP): garante que cada arquivo tenha um propósito único e bem definido.
• Melhoria na Leitura e Manutenção: o main.py fica reduzido apenas à função de bootstrap (inicialização) da API.
• Reuso e Testabilidade: schemas, loaders e routers podem ser importados e testados de forma independente.
• Escalabilidade: permite adicionar novas rotas ou versões da API sem sobrecarregar um único arquivo.
